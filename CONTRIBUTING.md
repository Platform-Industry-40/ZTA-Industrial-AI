# Contributing to ZTA Industrial AI

[![English](https://img.shields.io/badge/language-English-blue?style=flat-square)](#contributing-to-zta-industrial-ai)
[![Deutsch](https://img.shields.io/badge/Sprache-Deutsch-green?style=flat-square)](#beitrag-zum-zta-industrial-ai)

**Welcome!** We appreciate your interest in contributing to the ZTA Industrial AI project. This repository focuses on Zero-Trust Architecture for Industrial AI systems, emphasizing documentation, research, and proof-of-concept developments. As a work-in-progress, we encourage feedback, suggestions, or questions via GitHub Issues. Currently, there are no external contributors actively involved in the project, but we are open to input that enhances the documentation, adds translations, or provides insights into related standards and use cases.

## How to Contribute

While the core development is handled by the project's authors and coordinators, you can:
- Open Issues for feedback, bug reports, or enhancement suggestions.
- Discuss topics related to the project in Issues.

If you wish to propose changes:
1. **Fork the Repository**: Start by forking the repo to your own GitHub account.
2. **Create a Branch**: Use a descriptive branch name (e.g., `feature/add-new-use-case` or `fix/typo-in-doc`).
3. **Make Changes**: 
   - For documentation: Edit Markdown files in `/doc/de/` or `/doc/en/`.
   - Ensure bilingual consistency where applicable.
   - Follow Markdown best practices for readability.
4. **Commit Your Changes**: Use clear commit messages (e.g., "Add section on EU AI Act compliance").
5. **Open a Pull Request**: Describe your changes, reference any related issues, and ensure they align with the project's goals. Pull Requests will be reviewed by the core team.
6. **Code of Conduct**: Be respectful and collaborative. We follow the [Contributor Covenant](https://www.contributor-covenant.org/).

## Reporting Issues

- Use GitHub Issues to report bugs, suggest enhancements, or ask questions.
- Provide details like steps to reproduce, expected vs. actual behavior, and relevant context.

# Beitrag zum ZTA Industrial AI

[![English](https://img.shields.io/badge/language-English-blue?style=flat-square)](#contributing-to-zta-industrial-ai)
[![Deutsch](https://img.shields.io/badge/Sprache-Deutsch-green?style=flat-square)](#beitrag-zum-zta-industrial-ai)

**Willkommen!** Wir freuen uns über Ihr Interesse, zum ZTA Industrial AI-Projekt beizutragen. Dieses Repository konzentriert sich auf Zero-Trust-Architektur für industrielle KI-Systeme, mit Schwerpunkt auf Dokumentation, Forschung und Proof-of-Concept-Entwicklungen. Als Work-in-Progress laden wir Feedback, Vorschläge oder Fragen über GitHub Issues ein. Derzeit sind keine externen Mitwirkenden aktiv am Projekt beteiligt, aber wir sind an Input interessiert, der die Dokumentation erweitert, Übersetzungen hinzufügt oder Einblicke in verwandte Standards und Use Cases bietet.

## So tragen Sie bei

Während die Kernentwicklung von den Autoren und Koordinatoren des Projekts gehandhabt wird, können Sie:
- Issues für Feedback, Fehlerberichte oder Verbesserungsvorschläge eröffnen.
- Themen im Zusammenhang mit dem Projekt in Issues diskutieren.

Falls Sie Änderungen vorschlagen möchten:
1. **Forken Sie das Repository**: Forken Sie das Repo in Ihr eigenes GitHub-Konto.
2. **Erstellen Sie einen Branch**: Verwenden Sie einen beschreibenden Branch-Namen (z. B. `feature/neuer-use-case-hinzufuegen` oder `fix/typo-in-doc`).
3. **Änderungen vornehmen**: 
   - Für Dokumentation: Bearbeiten Sie Markdown-Dateien in `/doc/de/` oder `/doc/en/`.
   - Stellen Sie bilingual Konsistenz sicher, wo zutreffend.
   - Folgen Sie Markdown-Best-Practices für Lesbarkeit.
4. **Committen Sie Ihre Änderungen**: Verwenden Sie klare Commit-Nachrichten (z. B. "Abschnitt zu EU AI Act Compliance hinzugefügt").
5. **Erstellen Sie einen Pull Request**: Beschreiben Sie Ihre Änderungen, verweisen Sie auf relevante Issues und stellen Sie sicher, dass sie zu den Projektzielen passen. Pull Requests werden vom Kernteam überprüft.
6. **Verhaltenskodex**: Seien Sie respektvoll und kooperativ. Wir folgen dem [Contributor Covenant](https://www.contributor-covenant.org/).

## Issues melden

- Nutzen Sie GitHub Issues, um Fehler zu melden, Verbesserungen vorzuschlagen oder Fragen zu stellen.
- Geben Sie Details wie Reproduktionsschritte, erwartetes vs. tatsächliches Verhalten und relevanten Kontext an.
